Pareto-Optimal Benders Decomposition

For the p-Hub-Arc Problem — Mathematical Derivation & Implementation
Based on Magnanti & Wong (1981) • Ramamoorthy et al. (EJOR 2024)

1. Problem Setting: p-Hub-Arc

Given n nodes, select exactly p directed hub arcs from the n(n−1) candidates H = {(u,v) : u ≠ v}. Every OD pair (i,j) routes its flow through the cheapest open arc.

Binary variable: y_a ∈ {0,1} for all a ∈ H, with ∑ y_a = p Arc cost: c_ij(u,v) = W_ij · (D_iu + D_uv + D_vj) After sorting unique costs per (i,j): C¹_ij ≤ C²_ij ≤ ... ≤ C^K_ij (K distinct levels) L^k_ij = set of arcs at cost level k

2. Benders Master Problem

Introduce surrogates θij for each OD pair with Kij > 1:

min ∑_(i,j) θ_ij + ∑_(i,j): K=1 C¹_ij s.t. ∑_(a ∈ H) y_a = p θ_ij ≥ Benders cuts y_a ∈ {0,1}, θ_ij ≥ 0

At each integer node, a subproblem per OD pair checks if the cut θij ≥ actual routing cost, and if not, adds a Benders cut.

3. The Cut Value Function & Dual Subproblem

Define the cut value function for dual variables ν at any point y:

f(ν, y) = C¹ + ν_1 (1 − ∑_(a∈L¹) y_a) − ∑_(k=2)^K ν_k ∑_(a∈L^k) y_a
Standard Dual Subproblem DSij(ȳ):
z* = max_ν f(ν, ȳ) s.t. ν_k − ν_(k+1) ≤ C^(k+1) − C^k (dual feasibility) ν_k ≥ 0, k = 1,...,K
Benders cut added to master:    θ_ij ≥ f(ν*, y)    (using optimal ν* found above)

4. The Weakness Problem: Multiple Dual Optima

Why standard cuts can be weak:

The dual LP usually has many optimal solutions ν* all achieving the same z* at ȳ. Different ν* give different cuts:

θ_ij ≥ f(ν(1), y) vs. θ_ij ≥ f(ν(2), y)

Both valid, but one may be much tighter at future integer solutions. A weak cut barely restricts θ away from ȳ, causing more B&B nodes.

Small example: K=3, C = [2, 5, 9]
  • ν(1) = (3, 2, 0): cut θ ≥ 2 + 3(1−∑L¹y) − 2∑L²y   ← involves cost levels 1 and 2
  • ν(2) = (5, 0, 0): cut θ ≥ 2 + 5(1−∑L¹y)            ← only uses cost level 1
Both achieve z*=4 at ȳ. But ν(1) is tighter when L² arcs are open.

5. Magnanti-Wong Pareto Selection

5.1 Core Point

A core point y° lies strictly inside the master feasible region:

y°_a = p / |H| for all a ∈ H Satisfies: ∑ y°_a = p (feasible) and 0 < y°_a < 1 (strictly interior)

5.2 Two-Step Pareto Dual (original approach, 2 LP solves)

Pareto Dual Subproblem:

Step 1. Solve DSij(ȳ) → get z*

Step 2. Among all dual-optimal ν at ȳ, find the one maximising cut tightness at the core point y°:

max_ν f(ν, y°) ← maximise at core point s.t. f(ν, ȳ) = z* ← must stay optimal at ȳ ν_k − ν_(k+1) ≤ C^(k+1) − C^k ← dual feasibility ν_k ≥ 0
Cost: 2 LP solves per OD pair per node (original implementation)

The resulting cut is:

  • Valid — ν* is dual-feasible
  • Tight at ȳ — f(ν*, ȳ) = z*, same as standard
  • Pareto-dominant — strongest possible among all valid cuts at this node

6. The ε-Perturbation Trick: One LP Only

Instead of 2 LPs with a coupling constraint, combine into one objective:

max_ν f(ν, ȳ) + ε · f(ν, y°), ε > 0 small (e.g. 10−&sup6;)

6.1 Algebraic Simplification

f(ν,ȳ) + ε·f(ν,y°) = C¹ + ν_1(1 − ∑_L¹ ȳ) − ∑_(k≥2) ν_k ∑_L^k ȳ + ε[C¹ + ν_1(1 − ∑_L¹ y°) − ∑_(k≥2) ν_k ∑_L^k y°] = (1+ε)C¹ + ν_1 [(1+ε) − ∑_L¹ (ȳ_a + ε y°_a)] − ∑_(k≥2) ν_k [∑_L^k (ȳ_a + ε y°_a)]         constant                      |—————————————| = εC¹ + f(ν, ỹ) where ỹ_a = ȳ_a + ε y°_a
Key Result (the ε-perturbation identity):
max_ν [f(ν,ȳ) + ε·f(ν,y°)] = εC¹ + max_ν f(ν, ỹ) where ỹ_a = ȳ_a + ε y°_a (shifted point)
Consequence: Run the standard dual LP once at ỹ — you get the Pareto-optimal ν directly. 1 LP instead of 2.

Why it works:
  • ε small → primary term f(ν,ȳ) dominates → ν* is still dual-optimal at ȳ
  • Among all dual-optimal ν, the ε·f(ν,y°) term acts as a tiebreaker selecting the one with highest cut value at y°

6.2 After Solving at ỹ

The cut violation check must use the true z* at original ȳ (not the shifted ỹ):

z* = f(ν*, ȳ) = C¹ + ν*_1(1 − ∑_L¹ ȳ_a) − ∑_(k≥2) ν*_k ∑_L^k ȳ_a

6.3 Choosing ε

Practical: ε = 10−&sup6; (standard in Magnanti-Wong literature) Principled: ε < min_k (C^(k+1) − C^k) / max_range(f(ν, y°))

8.1 Single-LP Pareto Dual

# pareto_benders_hub_arc.py

def _solve_dual_subproblem_pareto(i, j, y_bar, y_core, C, L, K, eps=1e-6):
    Kij = K[(i, j)]
    L1  = L[(i, j)][1]
    C1  = C[(i, j)][0]

    # STEP 1: build shifted point  y_eff = y_bar + eps * y_core
    y_eff = {a: y_bar.get(a, 0.) + eps * y_core.get(a, 0.) for a in y_bar}

    # STEP 2: standard dual LP at y_eff  (ONE solve, no coupling constraint)
    m  = gp.Model(); m.Params.OutputFlag = 0
    nu = m.addVars(range(1, Kij+1), lb=0., vtype=GRB.CONTINUOUS)

    obj = C1 + nu[1] * (1. - sum(y_eff.get(a, 0.) for a in L1))
    for k in range(2, Kij+1):
        Lk = L[(i,j)].get(k, [])
        obj -= nu[k] * sum(y_eff.get(a, 0.) for a in Lk)
    m.setObjective(obj, GRB.MAXIMIZE)

    for k in range(1, Kij):                    # dual feasibility
        m.addConstr(nu[k] - nu[k+1] <= C[(i,j)][k] - C[(i,j)][k-1])
    m.optimize()

    nu_sol = {k: nu[k].X for k in range(1, Kij+1)}

    # STEP 3: recompute z* at ORIGINAL y_bar (not y_eff!)
    z_star = C1 + nu_sol[1] * (1. - sum(y_bar.get(a, 0.) for a in L1))
    for k in range(2, Kij+1):
        Lk = L[(i,j)].get(k, [])
        z_star -= nu_sol[k] * sum(y_bar.get(a, 0.) for a in Lk)

    cut = {"Ki": Kij, "nu": nu_sol, "C1": C1}
    for k in range(1, Kij+1): cut[f"L{k}"] = L[(i,j)].get(k, [])
    return z_star, cut

8.2 Lazy Callback (Phase 2 — fires at every integer node)

class ParetoBendersCutCallback:
    def __call__(self, model, where):
        if where != GRB.Callback.MIPSOL: return

        y_bar   = {a: model.cbGetSolution(self.y_vars[a])          for a in ...}
        theta_b = {(i,j): model.cbGetSolution(self.theta_vars[...]) for ...}

        # Pareto separation: internally uses y_eff = y_bar + eps*y_core
        cuts, _ = separation_pareto(y_bar, theta_b, self.y_core, ...)

        for ((i,j), cut) in cuts:
            nu   = cut["nu"]
            expr = cut["C1"]
            expr += nu[1] * (1. - gp.quicksum(self.y_vars[a] for a in cut["L1"]))
            for k in range(2, cut["Ki"]+1):
                expr -= nu[k] * gp.quicksum(self.y_vars[a] for a in cut[f"L{k}"])
            model.cbLazy(self.theta_vars[(i,j)] >= expr)  # add Pareto cut

# p arcs selected from |H| = n(n-1) candidates
# Uniform interior share: sum = p, each value in (0,1)
